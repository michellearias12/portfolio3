Estandarización de cálculos de disponibilidad contractual para 50 plantas

Revisé y unifiqué la lógica contractual de disponibilidad de 50 plantas renovables, cada una con su propia metodología, para convertir decenas de fórmulas heterogéneas en un conjunto de 7 fórmulas estandarizadas y parametrizables, listas para integrarse en una plataforma digital de gestión de activos.

Resumen ejecutivo

  • Rol: Data Analyst / Diseñadora de producto
  • Contexto: contratos y lógica de disponibilidad en activos renovables
  • Enfoque: análisis documental, estandarización, parametrización y diseño para producto
  • Resultado: un módulo flexible, escalable y consistente para cálculo automático
El problema

Cada planta hablaba un idioma contractual distinto

La disponibilidad contractual se calculaba de formas muy diferentes entre plantas, lo que dificultaba automatizar, comparar y reportar resultados de forma confiable dentro de una sola plataforma.

Cada contrato definía la disponibilidad con reglas distintas: unos medían por tiempo, otros por potencia, otros mezclaban ambos criterios o excluían eventos de manera particular. Eso hacía inviable escalar el cálculo sin revisar cada caso manualmente.

  • 50 plantas con documentos y metodologías personalizadas.
  • Fórmulas distintas por nivel: equipo, instalación, mensual, anual o diario.
  • Dificultad para comparar activos dentro del mismo portafolio.
  • Alto esfuerzo manual y mayor riesgo de inconsistencias en reportes.
Mi rol

Traducir complejidad contractual en una lógica estándar de producto

Mi trabajo consistió en revisar los contratos, extraer la lógica relevante, detectar patrones y transformarlos en una estructura común que pudiera vivir dentro de una plataforma digital sin perder flexibilidad.

1. Revisión y extracción documental

Analicé uno por uno los documentos contractuales de 50 plantas para identificar fórmulas, reglas, exclusiones, periodos de cálculo y niveles de aplicación de la disponibilidad contractual.

2. Síntesis analítica

Detecté redundancias, variaciones y casos particulares para separar lo realmente único de lo que era parametrizable.

3. Diseño orientado a producto

Convertí esa diversidad en un set limitado de fórmulas estándar con parámetros configurables, listas para integrarse a una solución digital escalable.

Qué demuestra este proyecto
  • Análisis profundo: capacidad de leer, interpretar y estructurar lógica compleja.
  • Pensamiento sistémico: habilidad para convertir excepciones en reglas escalables.
  • Diseño de producto: visión para que la solución no sea solo correcta, sino usable e integrable.
  • Rigor analítico: equilibrio entre precisión contractual y simplicidad operativa.
La solución

De decenas de fórmulas a un sistema de 7 fórmulas parametrizables

En lugar de tratar cada planta como un caso aislado, diseñé una lógica común con parámetros configurables que permite adaptar el cálculo sin reinventarlo desde cero.

Puedes reemplazar este bloque con capturas reales de la plataforma, dashboards o tablas de parametrización.

Estandarización de fórmulas

Reduje múltiples metodologías contractuales a 7 fórmulas estándar capaces de cubrir todos los escenarios relevantes mediante configuración.

Parametrización flexible

Definí variables como periodo de cálculo, nivel de agregación, tipo de potencia, umbrales y exclusiones para adaptar la lógica a cada contrato.

Preparación para implementación

Documenté la lógica de forma clara para que pudiera desarrollarse como módulo dentro de la plataforma de gestión de activos renovables.

Lineamientos de visualización

Propuse cómo presentar resultados en front-end: vistas mensuales, acumulados anuales, por equipo o por planta, así como opciones de configuración para el usuario.

Lógica uniforme

El cálculo deja de depender de interpretaciones manuales y pasa a operar bajo una lógica estructurada, consistente y repetible.

Escalabilidad real

Una nueva planta puede integrarse configurando parámetros, sin tener que diseñar un cálculo completamente nuevo.

Diseño listo para producto

La solución quedó preparada no solo para análisis, sino para convertirse en funcionalidad dentro de una plataforma digital.

Impacto

Automatización, comparabilidad y menos trabajo manual

El proyecto permitió transformar un portafolio heterogéneo en una lógica operable, comparable y mucho más fácil de mantener dentro de una plataforma digital.

Cálculo automático y consistente

La plataforma puede calcular disponibilidad contractual de cualquier planta sin depender de procesos manuales distintos para cada contrato.

Benchmarking entre activos

Al estandarizar la lógica, se facilita comparar plantas, monitorear cumplimiento y generar reportes homogéneos para todo el portafolio.

Onboarding más rápido

Integrar nuevas plantas se vuelve mucho más simple, porque la lógica ya existe y solo necesita parametrizarse según el contrato correspondiente.

En una frase

Este proyecto consistió en pasar de 50 maneras distintas de medir disponibilidad a una sola arquitectura lógica capaz de absorber complejidad sin perder claridad, precisión ni escalabilidad.
